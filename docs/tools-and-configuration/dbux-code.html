Runtime Serverâ€‹

Currently, the Dbux VSCode Extension offers the only implementation of the Dbux ðŸ“¡ Runtime Server. If enabled, it receives data from applications running on localhost1. Once a first batch of data is received, the application will show up in the Application View.

The Runtime Server turns on automatically the first time you use the Run Button or run a Dbux Practice exercise. In any other case, you need to turn it on manually. The button is located at the top of the Applications view2:

Also see: Dbux Architecture.
